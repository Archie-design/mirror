## 1. 遊戲核心理念

**主題**：以綠野仙蹤旅程為敘事框架，讓每位學員成為自己人生劇本的主角，踏上屬於自己的黃磚路。

**故事主題**：**你的黃磚路** — 綠野仙蹤 × 親證班
以《綠野仙蹤》為敘事框架，五位旅伴角色對應五運方向，學員踏上屬於自己的黃磚路旅程。親證班不是給你一個你沒有的東西，而是帶你走回那個你早已擁有卻遺忘的自己。

**四大活動方向**：
1. 提升人事物關係
2. 找到道命夥伴
3. 五運目標親證
4. 培養向上慣性

**活動期間**：2026年5月10日 ～ 7月12日（共9週）

### 1.1 修練旅途與旅伴系統

每位學員在修練旅途中將選擇一位**旅伴**（對應事業、財富、情感、家庭、體能五運之一），並根據自身最需要突破的方向，完成對應的**人生大戲九宮格**任務。

旅伴角色選擇邏輯同「無限體系」。

| 五運 | 旅伴名稱 | 原型 | 核心象徵 | 個性特質 | 故事意涵 |
|------|---------|------|---------|---------|---------|
| 體能運 | 小桃 | 桃樂絲（主角） | 回到自己・身體是旅程的起點 | 純粹、勇敢、從不放棄回家的信念；不是最強，但最清楚自己想要什麼 | 龍捲風象徵突如其來的身體警訊。健康是所有旅程的根基，小桃腳上的紅鞋象徵身體本來就有的自癒力，只是我們往往忘記它一直都在 |
| 事業運 | 小草 | 稻草人 | 智慧與判斷力・以為自己不夠聰明 | 充滿好奇心、熱愛思考，常自我懷疑，但關鍵時刻永遠是他想到辦法 | 很多人在事業上不是沒有能力，而是不相信自己的判斷。小草整趟旅程出點子最多，幫學員看見本來就有的智慧與行動力 |
| 情感運 | 小鐵 | 鐵皮人 | 愛與流動・以為自己沒有心 | 細膩、敏感、對夥伴極度在意。表面用鐵殼包裹，內心卻最柔軟 | 感情的課題從來不是找到對的人，而是先打開自己的心。你給出什麼能量，就吸引什麼樣的關係回來 |
| 財富運 | 小獅 | 膽小獅 | 勇氣與值得感・以為自己不夠格 | 外表威武，內心充滿恐懼與自我懷疑。但每次危機，第一個站出來的永遠是他 | 財富的阻礙往往不是方法不對，而是內心覺得自己不值得、不敢承擔、不敢被看見 |
| 家庭運 | 翡翠城 | 翡翠城（目的地） | 歸屬感・所有旅程最終的終點 | 神秘、強大、令人嚮往。最終揭示的真相：真正的力量從來不在城裡，而在每個人自己身上 | 家庭是所有人能量的根源。很多人的低能量根源藏在未修復的家庭關係裡，翡翠城象徵「我想回去的地方」 |

> **UI 備註**：翡翠城雖為地點而非人物，在旅伴選擇卡片上與其他四位同等呈現（相同尺寸卡片），以城堡圖示區別，搭配說明文字「象徵你回家的渴望」。角色參考圖像：`reference/S__33480706.jpg`。

### 1.2 九宮格親證系統

學員在「人生大戲」任務中，針對選定的最弱一運，完成**親證九宮格**：

- 九宮格共9個格子，內容依學員選定的**旅伴（五運角色）**從公版模板套用
- 每完成一格即打卡記錄，**格子本身不計分**（僅記錄完成狀態）
- 九宮格連線成功（一條 = 3格連線）：立即獲得額外 **+300分**
- 連線上限：8條（最高額外 **+2400分**）
- **人生大戲分享**（wk4_small / wk4_large）整合於人生大戲分頁，需**當週至少完成一格**後方可打卡分享；系統會自動阻擋未達條件的打卡，每週同類型也僅能入帳一次

#### 公版模板系統

系統為五種旅伴各提供一份**公版九宮格模板**（共5份），對應五運方向：

| 旅伴類型 | 角色名稱 | 說明 |
|---------|---------|------|
| 體能運旅伴 | 小桃 | 健康・回到自己方向親證格子 |
| 事業運旅伴 | 小草 | 事業・智慧與判斷力方向親證格子 |
| 情感運旅伴 | 小鐵 | 感情・愛與流動方向親證格子 |
| 財富運旅伴 | 小獅 | 財富・勇氣與值得感方向親證格子 |
| 家庭運旅伴 | 翡翠城 | 家庭・歸屬感方向親證格子 |

- 學員選定旅伴後，系統自動套用對應公版模板至個人九宮格
- 學員本人**唯讀**，不可自行修改格子內容；格子內容固定使用公版模板

### 4.5 小組凝聚（實體）QR 流程

**六步流程**：

1. **管理員排期** — 在「大隊長總部」→「安排實體凝聚」選定某小隊與凝聚日期。
2. **小隊長取得 QR** — 凝聚當日登入「隊長基地」，系統顯示該場次的 QR Code；隊員與大隊長掃碼即可報到。
3. **隊員／大隊長掃碼** — 掃碼進入報到頁，點「確認到場」完成報到。
4. **小隊長送出初審** — 當日結束後按「送出審核」，之後報到名單凍結、無法再掃碼。
5. **大隊長終審** — 在「大隊長總部」→「待審實體凝聚」核准或退回；退回時可填備註。
6. **批次入帳** — 核准通過後，系統自動依照出席狀況替每位出席成員入帳。

**計分規則（每位出席成員）**：

```
基礎出席：                        +300
├─ 若小隊全員到齊：              +100
└─ 若大隊長本人也到場：          +100
                                 ─────
最高：                           500 / 人
```

- **全員到齊的分母**：該小隊所有在冊成員；掃碼人數達成此數即觸發。
- **大隊長到場加分**：只要有任一位大隊長掃碼即觸發（可與全員到齊疊加）。

**安全設計（為什麼能防止作弊）**：

| 風險 | 防線 |
|--------|------|
| QR 被轉發到群組、遠端簽到 | 單日有效；系統自動比對掃碼者是否屬於該小隊（或為大隊長） |
| 小隊長偽報全員到齊 | 入帳依實際掃碼紀錄計算，小隊長無法代簽 |
| 學員冒充大隊長身分 | 「大隊長到場」以系統內該帳號的大隊長角色為準，無法由掃碼端指定 |
| 冒用他人身分掃碼 | 所有操作需以本人登入狀態進行 |
| 重複入帳 | 一場次每位成員只記一筆出席，核准後即鎖定 |

### 4.6 小組凝聚（線上）一級審核流程

**三步流程**：

1. **學員提交** — 在「旅伴週報」→「小組凝聚」卡片點「提交本週凝聚申請」，可附備註。
2. **小隊長初審** — 小隊長在「隊長基地」→「本週線上凝聚審核」看到本隊待審清單：
   - 核准 → 系統即時入帳 +100 分。
   - 退回 → 狀態標記為已退回，可填備註；學員當週可修正後重新提交。
3. **直接入帳** — 小隊長核准即完成，無需大隊長終審。同一學員同一週不會重複入帳。

**安全設計**：

| 風險 | 防線 |
|--------|------|
| 學員偽造「本週」日期 | 「本週」以系統時區（Asia/Taipei）自動判定，無法由學員端指定 |
| 同週重複提交 | 同一學員同一週只允許一筆進行中或已核准的申請 |
| 小隊長審核到別隊申請 | 小隊長只能審核本小隊的申請 |
| 冒用他人身分提交／審核 | 所有操作需以本人登入狀態進行，小隊長身分由系統驗證 |
| 重複入帳 | 同一學員同一週只會成功入帳一次 |

### 5.1.1 大隊長分數計入小隊／大隊平均的方式

大隊長不歸入任何單一小隊，但其全額分數會同步計入所屬大隊下的**每一個小隊**：

- `小隊平均分 =（小隊成員分數總和 + 所有大隊長分數之和）÷（小隊成員人數 + 大隊長人數）`
- 考量有些大隊有兩位大隊長，若大隊有一到兩位大隊長統轄 N 個小隊，N 個小隊各自計算時都多加一到兩位「大隊長成員」（依實際大隊長人數）；每位大隊長本人的分數不會被稀釋。
- 大隊層級排名依相同精神聚合：大隊總分 = Σ(小隊成員分數) + Σ(大隊長分數)；大隊人數 = Σ(小隊成員人數) + 大隊長人數（1–2人）。

**設計理由**：大隊長的修練狀態象徵全大隊的風向；讓每個小隊在排名時共享大隊長分數，可激勵小隊與大隊長合作成長，而非把大隊長排除在小隊榮譽體系之外。
